Measures Pipeline Guide

Build multi-step data calculations in ytics — no code required.

What is the Measures Pipeline?

The Measures Pipeline lets you define multi-step data transformations directly on a chart widget. Instead of being limited to "group by X, aggregate Y," you can chain operations to answer complex questions like:

  • "What is the average salary in the most populated city of each country?"
  • "Show the top 3 products by revenue for each region."
  • "Compare total sales only for stores that exceeded 1 million in revenue."

Each step transforms the data, and the next step operates on the output of the previous one — like a pipeline.

The 5 Step Types

⊞ Group & Aggregate

Collapses many rows into fewer rows by grouping, then computes summary values per group.

SettingDescription
Group by columnsOne or more columns to group by (hold Ctrl/Cmd for multi-select)
AggregationsFor each group, compute values: Sum, Count, Average, Min, Max, Median, Std Dev, percentiles. Each aggregation has a source field, a function, and an output name ("as").
⊤ Top / Bottom N

Keeps only the top (or bottom) N rows, optionally within each group.

SettingDescription
NNumber of rows to keep (e.g., 1, 3, 10)
Order byThe column to rank by
DirectionTop (descending) or Bottom (ascending)
Per groupOptional. If set, picks top N within each group independently.
⊘ Filter

Removes rows that don't match a condition.

SettingDescription
ColumnThe column to test
Operator= != > < >= <= contains is null …
ValueThe comparison value
ƒ Compute Column

Creates a new column from a JavaScript expression using existing column names as variables.

SettingDescription
New column nameName for the computed column (e.g., salary_ratio)
ExpressionA formula: avg_salary / cost_of_living, population > 1000000 ? "large" : "small"
↕ Sort

Reorders rows by a column value.

SettingDescription
Sort byColumn to sort on
DirectionAscending or Descending

How to Access

  1. In Developer Mode, click on any chart widget (or create a new one).
  2. The widget editor sidebar opens. You'll see tabs: Fields, Measures, Aesthetics, Options.
  3. Click the Measures tab.
  4. Add steps using the buttons at the bottom: ⊞ Group & Aggregate ⊤ Top / Bottom N etc.
  5. Configure each step by clicking its header to expand it.
  6. Use Preview output at the bottom to inspect intermediate results.
  7. Go back to the Fields tab — the column dropdowns now show pipeline output columns instead of raw dataset columns.
Tip: Backward compatible

If the Measures pipeline is empty (no steps), charts work exactly as before — using their built-in aggregation from the Fields tab. You only need the pipeline for multi-step calculations.

Example 1: Average Salary in the Most Populated City per Country

Question: "For each country, find the most populated city, then show that city's average salary in a bar chart."

Step 1 — Group & Aggregate

Group by: country, city
Aggregations: Sum of population → total_pop, Average of avg_salary → salary

Step 2 — Top 1 per Country

N: 1 · Order by: total_pop · Direction: Top (desc) · Per group: country

Step 3 — Sort

Sort by: salary · Direction: Descending

After pipeline, set in Fields tab: X Axis = country, Value = salary

Example 3: Salary-to-Cost Ratio for Large Cities

Question: "Among cities with over 1 million population, which have the best salary-to-cost-of-living ratio?"

Step 1 — Filter

Column: population · Operator: > · Value: 1000000

Step 2 — Compute Column

Name: salary_ratio · Expression: avg_salary / (cost_of_living * 12)

Step 3 — Sort

Sort by: salary_ratio · Direction: Descending

Step 4 — Top 15

N: 15 · Order by: salary_ratio · Direction: Top

Fields tab: X = city, Value = salary_ratio

Key Concepts

Pipeline is per-widget

Each chart has its own independent pipeline. Two bar charts using the same dataset can have completely different measure pipelines. This is different from Dataset Transforms (in the Data Integration tab), which apply globally to all widgets sharing that dataset.

Empty pipeline = classic behavior

If you don't add any pipeline steps, the chart works exactly as before — it uses its built-in aggregation (the Aggregation dropdown in the Fields tab). The pipeline is purely opt-in.

Output columns replace raw columns

Once you add pipeline steps, the Fields tab column dropdowns automatically switch to show the pipeline's output columns. For example, after a Group & Aggregate step that outputs country and total_pop, those are the only columns available for X/Y mapping.

Preview before committing

Click "Preview output" at the bottom of the Measures tab to see a sample table of what the pipeline produces. This runs on the first 100 rows of your dataset for quick feedback.

Step ordering matters

Steps execute top-to-bottom. A Filter after a Group & Aggregate filters the aggregated rows, not the raw data. Use the ↑↓ arrows to reorder steps.

Common pattern: Group → Top N → Chart

Most complex questions follow this pattern:
1. Group & Aggregate to summarize data at a finer level than you need.
2. Top/Bottom N (per group) to pick the representative row per group.
3. Optionally Sort or Filter the result.
4. Map the output columns to chart fields.
